█ DEPLOYMENT STEPS

  LOCAL DEVELOPMENT (30 minutes):
    □ pnpm dev:all  ──►  Runs everything together
    □ Test login at http://localhost:3000
    □ Verify API at http://localhost:8000/api

  VERCEL DEPLOYMENT (20 minutes):
    □ Go to vercel.com
    □ Import GitHub repo (allenamaya/Kenwell-ERP-System)
    □ Deploy with one click
    □ Note your Vercel URL

  RAILWAY DEPLOYMENT (25 minutes):
    □ Go to railway.app
    □ Import GitHub repo
    □ Add PostgreSQL service
    □ Set environment variables
    □ Note your Railway URL

  CONNECT THEM (5 minutes):
    □ Update Vercel env: NEXT_PUBLIC_API_URL=<railway-url>
    □ Update Railway CORS: Your Vercel URL
    □ Test login works end-to-end


█ ENVIRONMENT VARIABLES

  Frontend (.env.local):
    NEXT_PUBLIC_API_URL=http://localhost:8000/api

  Frontend (Vercel):
    NEXT_PUBLIC_API_URL=https://your-backend.railway.app/api

  Backend (.env):
    DEBUG=True/False
    SECRET_KEY=your-generated-key
    DATABASE_NAME=kenwell_db
    DATABASE_USER=postgres
    DATABASE_PASSWORD=password
    CORS_ALLOWED_ORIGINS=https://your-app.vercel.app


█ HELPFUL COMMANDS

  Local Development:
    pnpm dev:all          ──► Run both frontend + backend
    pnpm dev:frontend     ──► Frontend only
    pnpm dev:backend      ──► Backend only

  Backend Setup:
    cd backend
    python -m venv venv
    source venv/bin/activate
    pip install -r requirements.txt
    python manage.py migrate
    python manage.py runserver

  Git Workflow:
    git add .
    git commit -m "your message"
    git push origin main
    └─► Auto-deploys to Vercel + Railway!

█ TROUBLESHOOTING QUICK FIXES

  "CORS error"
    └─ Update CORS_ALLOWED_ORIGINS in Django

  "401 Unauthorized"
    └─ Check JWT token is being sent

  "Database connection failed"
    └─ Verify DATABASE_* env variables

  "API not responding"
    └─ Check backend is running and URL is correct

  "Frontend won't load"
    └─ Check NEXT_PUBLIC_API_URL in env variables

  See VERCEL_DEPLOYMENT_GUIDE.md for more troubleshooting
